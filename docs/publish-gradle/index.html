üß™ Reusable GitHub Workflow: publish-gradle.yml

This GitHub Actions workflow is designed to automatically publish a Gradle project to GitHub Packages every day, using a versioning format based on the current date.

üîÅ Purpose

  • Reusable and callable from multiple repositories
  • Calculates version based on UTC date
  • Validates all mcengine(-name)-(api|common).version entries in gradle.properties
  • Fails if version format does not match expected
  • Publishes to GitHub Packages with Gradle

üß™ Example Usage

Use this in another repo to call the reusable workflow:

jobs:
  publish:
    uses: MCEngine/actions/.github/workflows/publish-gradle.yml@master
    with:
      repo-name: ${{ github.repository }}
    secrets:
      GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}

This will call the centralized workflow hosted at MCEngine/actions.

üì¶ Repository Requirements

  • Include a valid gradle.properties with version keys
  • Configure publishing block in build.gradle to use GitHub Packages

    • GITHUB_USERNAME = ${{ github.actor }}
    • GITHUB_TOKEN = ${{ secrets.GITHUB_TOKEN }}
